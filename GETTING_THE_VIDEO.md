## Step 1: Check if Video is Ready

Visit Google Cloud Console:
https://console.cloud.google.com/vertex-ai/generative/video/locations/us-central1?project=character-ai

You should see your video operation. It takes **2-5 minutes** to complete.

## Step 2: Download the Video

### Option A: From Cloud Console (Easiest)

1. Click on your operation in the console
2. When complete, you'll see a download link or GCS URI
3. Download the video file

### Option B: Using gsutil (if GCS URI provided)

```bash
# If the video is at gs://some-bucket/path/video.mp4
gsutil cp gs://bucket-name/path/to/video.mp4 ~/Downloads/scene-1.mp4
```

## Step 3: Add Video to Website

### Using the Helper Script

```bash
cd ~/Documents/parable

# From GCS URI:
./scripts/download-video.sh gs://bucket-name/path/to/video.mp4

# Or from local file:
./scripts/download-video.sh ~/Downloads/scene-1.mp4
```

This will copy the video to `public/videos/scene-1.mp4`.

### Or Manually

```bash
cd ~/Documents/parable
mkdir -p public/videos
cp ~/Downloads/your-video.mp4 public/videos/scene-1.mp4
```

## Step 4: Deploy to Website

```bash
cd ~/Documents/parable
npm run deploy
```

The video will appear on: https://camptracker.github.io/parable/#/screenplay

## Troubleshooting

### Video Not Showing After Deploy

1. **Hard refresh** the website: Cmd+Shift+R
2. **Check file exists:**
   ```bash
   ls -lh public/videos/scene-1.mp4
   ```
3. **Verify deployment:**
   ```bash
   curl -I https://camptracker.github.io/parable/videos/scene-1.mp4
   ```

### Can't Find Video in Console

The operation might still be processing. Veo 2.0 typically takes:
- **Minimum:** 2 minutes
- **Average:** 3-4 minutes  
- **Maximum:** 10 minutes

Check back in a few minutes and refresh the console page.

### GCS URI Not Working

Make sure you have `gsutil` installed and authenticated:
```bash
gcloud auth login
gsutil ls  # Should list buckets
```
